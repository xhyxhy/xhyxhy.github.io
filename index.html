vscode配置全栈开发环境

发表于 2020-11-02 更新于 2020-11-03 分类于 vscode

最近因为要写shell，就在vscode上配置了shell的环境，发现vscode现在支持的插件足以满足各种语言开发环境，可以短暂的告别idea，记录下vscode相关的配置和插件。

工作区是什么：

vscode里一开始是以文件夹为形式打开项目，后来又新出了工作区的概念，工作区可以理解为把多个项目放到一起的地方（一种组织项目的形式，可以在工作区配置独特的配置，比如关闭某些插件）

常用插件：

UI类

Atom One Dark Theme：更改主题的插件，下载后可以在.vscode/theme文件里对主题配色修改。
Bracket Pair Colorizer：不同括号用不同颜色表示，和idea的Bracket一样。

java：

Java Extension Pack：运行和debug java的扩展包

翻译：

Chinese：汉化
Comment Translate：翻译软件，类似idea上的translate

通用：

Code Runner：给各个语言提供运行支持（但需要自己先配好功能），这个插件实际上就是给编辑器页面提供了运行按钮，你点了运行按钮后，插件就会去插件里的json配置寻找相应后缀关联的exe，用命令行来写出来，实际上就是这个插件帮你用命令行来运行程序。
json：对json格式化并且在右侧边栏为json生成json树。
Surround With：将代码块用try catch包围
JVM Bytecode Viewer：查看java字节码文件
koroFileHeader：生成代码头部注释或者方法注释
Local History：类似idea内置的local history
Paste JSON as Code：JSON生成对应代码类
Remote-WSL：连接WSL插件（真香，也是转战vscode的主要原因）

shell：

shell-format：格式化shell
shellcheck：shell代码静态检查
shellman：代码补全和高亮

format：

Prettier：格式化前端代码
Prettify JSON：格式化JSON，和json插件有重复

lint：

SonarLint：前端lint工具

linux总结

发表于 2020-11-01 更新于 2020-11-03 分类于 Linux

一.shell基本概念

1）终端和shell的区别

终端指用户连接的媒介，即接收用户输出并且显示结果的地方

shell指操作系统提供的解释器，shell接收终端的输入进行相应的处理，并将结果显示到终端上

2）文件系统

如下是每个发行版的根目录

常用的有如下目录：
/bin: 系统启动和运行的一切文件
/dev：内核识别设备后，将其放在这个文件夹
/etc：系统和各类程序的配置文件
/home：用户目录
/lib： 核心系统程序使用的共享库文件，类似DDL
/tmp： 系统临时文件
/usr： 各类可执行程序所在地。如经常使用的ls、cat命令等都在这个文件夹
/var： 可能改变的数据，比如用户邮件等

3）识别命令

type：显示命令的类型，命令分四种类型：
　　　１.可执行程序，即在usr中的命令
　　　２.内置命令
　　　３.shell函数
　　　４.alias命令
which：显示可执行程序的位置
help man：获取内置命令的帮助文档

4）重定向和管道

程序的输入输出默认都是在标准输入和输出，如果想改变输入和输出的来源，可以使用重定向
>> << > <意思是改变输入输出的来源至XX，可以为文件等
管道：用cmd1|cmd2表示，即cmd1命令的输出作为cmd2命令的输入

5）扩展

即在执行命令前（或者在脚本里）将某些内容扩展为其他内容，如echo　*，会输出当前目录下索引文件的名称，因为在执行命令前，shell会自动匹配当前目录下的文件，即echo *被扩展为echo “所有的文件名”

6）权限和用户管理

一个文件的权限属性用9个字母表示，三个一组分别代表用户权限，组权限，其他权限，创建文件时根据umask码对文件设置权限，可用chmod命令更改文件权限

7）进程

ps命令：当前进程快照
top命令：动态查看当前进程
&+程序可使程序在后台运行，fg命令可使程序回到前台

8） 环境

当一个用户登录时，shell就会读取对应的配置文件来配置环境变量
non-login shell： ~/.bash_profile or ~/.bash_login or ~/.profile
login shell： ~/.bashrc
可在文件里配置环境变量（如配置PS1来修改shell提示符，内置shell函数来实现自定义命令）

9) 网络

ping：测速网络连通性和速度
traceroute：跟踪网络数据包的传输数据
netstat：查看网络连接状态和端口等

10）文件查找和备份

find：支持正则匹配并且多种选项对查找到的文件进行操作
tar：备份
gunzip、zip：压缩

11）文本处理

（1）sort uniq对文本内容进行排序和去重
（2）cut paste jon来对文本进行切片处理
（3）comm diff来对文本进行比较处理
（4）sed 对文本进行替换等操作（非常强大的处理程序）

二.shell脚本基础

概念：shell脚本是包含一系列命令的文件，shell读取这个文件，执行这些命令



here文档：用<<token

>>token来表示，意思token中间的内容会被命令当成文件


流程控制中的true和false：

其实shell里并没有if true这种概念，而是通过命令的exitstatus来实现，如果一个程序执行成功了就会返回0。shell里有一个test命令，其接收一个bool值，如果为true返回0。if只接收exitstatus，所以if [[逻辑表达式]]（[[]]表示tet命令）的意思是if （test 表达式）从而实现流程控制


debug：

可以在在脚本中加入set -x来打印每步命令执行的信息，也可自己在脚本中echo相应的信息来调试。



位置参数：

在命令行接收的参数可用 $n来表示



管道陷阱：

echo “foo” |read
echo $REPLY
实际上上的命令并不会输入任何值，因为linux会打开一个子shell，将cmd1的输出到子shell的输入，在子shell执行read，执行后回到父shell，因为REPLY是在子shell中赋值的，所以在父shell中并没有值。可以使用组命令来解决这个问题。



多进程控制：

trap cmd signal程序可以接收某个signal来执行cmd从而实现脚本的通信和控制。
mkfifo可以创建管道，进程通过管道通信 。
wait pid可以实现进程同步。
